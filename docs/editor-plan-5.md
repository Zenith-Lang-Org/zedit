# Zedit Phase 5 — Deep Performance from microsoft/edit

> Analysis date: 2026-02-25. Based on direct source comparison between
> zedit and `microsoft/edit` (MIT). All improvements use `std` + libc FFI only.
> No external crate dependencies introduced.

Six targeted improvements ordered strictly from lowest to highest implementation
complexity. Each phase is self-contained and can be merged independently.

### Rationale

**Phase 31 first** — Pure math changes inside `src/oklab.rs`. No API changes, no new
files. Replaces one slow transcendental call (`powf(2.4)`) with a table lookup and one
`cbrt()` with a 4-operation bit hack. Risk is zero: tests already cover the math.

**Phase 32 second** — Adds ~80 lines to `src/glob.rs`. A pure additive fast path; the
slow path is untouched. Covers the 90% case (`**/*.ext`, `**/name`) with a simple
suffix match that skips the two-pointer algorithm entirely.

**Phase 33 third** — Adds a 256-slot hash table inside `src/render.rs` for color
memoization. Isolated to the rendering module. The most common scenario (few distinct
colors in a theme) turns repeated OKLab computation into a single comparison.

**Phase 34 fourth** — New `src/simd.rs` file. Replaces the 8-byte SWAR word scan in
`scan_newlines()` with AVX2 (x86_64), NEON (aarch64), or SWAR fallback, dispatched
at runtime. Touches `src/buffer.rs` in one function only.

**Phase 35 fifth** — Modifies the line cache architecture in `src/buffer.rs` from a
dense `Vec<usize>` of every line start to a sparse checkpoint layer (one entry per
8 192 lines). Requires Phase 34 to be useful (scan from checkpoint uses SIMD). Reduces
memory from O(n_lines) to O(n_lines / 8192).

**Phase 36 last** — The deepest architectural change. Replaces `Vec<u8>` as the gap
buffer backing store with a virtual-memory region (anonymous mmap + mprotect commit),
eliminating the ~50 MB practical ceiling for editable files. Requires a new
`src/vmem.rs` module and careful integration with `Buffer`. Highest risk.

## Phase 31: OKLab LUT + Fast Cube Root

**Goal**: Replace the two slowest operations in `srgb_to_oklab()` — the `powf(2.4)`
gamma expansion and the `f32::cbrt()` calls — with an O(1) lookup table and a
fast bit-manipulation estimate respectively. Add the reverse transform for future
use (color blending in OKLab space).

### Problem

`src/oklab.rs` current hot path:

```rust
fn srgb_to_linear(c: f32) -> f32 {
    if c <= 0.04045 {
        c / 12.92
    } else {
        ((c + 0.055) / 1.055).powf(2.4)   // ← transcendental: ~20 ns each call
    }
}

// In srgb_to_oklab():
let l = l.cbrt();   // ← std cbrt: ~5 ns
let m = m.cbrt();
let s = s.cbrt();
```

`srgb_to_oklab` is called for every distinct color during rendering:
once per token span per visible line per frame. At 60 FPS with a busy source
file, this accumulates. Each `powf(2.4)` is a transcendental function call;
three `cbrt()` per color add up similarly.

ms/edit solves both:
1. A compile-time `[f32; 256]` LUT indexed by the raw `u8` channel value —
   `srgb_to_linear` becomes an array lookup.
2. A fast cube-root estimator via the float bit-manipulation trick (same family
   as the famous fast inverse square root) plus one Newton–Raphson step, accurate
   to < 6.7 × 10⁻⁴ — sufficient for color work.

```rust
/// Fast cube-root estimator: bit-manipulation seed + 1 Newton step.
/// Max error < 6.7e-4 for inputs in [0, 1]. That's 0.067% — fine for color.
/// Reference: http://metamerist.com/cbrt/cbrt.htm (same trick as ms/edit)
#[inline(always)]
fn fast_cbrt(a: f32) -> f32 {
    if a == 0.0 { return 0.0; }
    let bits: u32 = a.to_bits();
    let est = f32::from_bits(bits / 3 + 709_921_077);
    // One Newton–Raphson step: x1 = (1/3) * (a/x0² + 2*x0)
    (1.0 / 3.0) * (a / (est * est) + est + est)
}
```

```rust
/// Convert OKLab (L, a, b) back to sRGB bytes (0–255 each), clamped.
pub fn oklab_to_srgb_u8(lab_l: f32, lab_a: f32, lab_b: f32) -> (u8, u8, u8) {
    let l_ = lab_l + 0.396_337_78 * lab_a + 0.215_803_76 * lab_b;
    let m_ = lab_l - 0.105_561_35 * lab_a - 0.063_854_17 * lab_b;
    let s_ = lab_l - 0.089_484_18 * lab_a - 1.291_485_55 * lab_b;

    let l = l_ * l_ * l_;
    let m = m_ * m_ * m_;
    let s = s_ * s_ * s_;

    let r = (4.076_741_66 * l - 3.307_711_59 * m + 0.230_969_93 * s).clamp(0.0, 1.0);
    let g = (-1.268_438_00 * l + 2.609_757_40 * m - 0.341_319_40 * s).clamp(0.0, 1.0);
    let b = (-0.004_196_09 * l - 0.703_418_61 * m + 1.707_614_70 * s).clamp(0.0, 1.0);

    (linear_to_srgb(r), linear_to_srgb(g), linear_to_srgb(b))
}

fn linear_to_srgb(c: f32) -> u8 {
    let v = if c > 0.003_130_8 {
        1.055 * c.powf(1.0 / 2.4) - 0.055
    } else {
        12.92 * c
    };
    (v * 255.0).round().clamp(0.0, 255.0) as u8
}
```

## Phase 32: Glob Fast Path

**Goal**: Add a constant-time fast path for the two patterns that account for ~90% of
real-world glob usage in file trees: `**/*.ext` and `**/filename`. Skip the two-pointer
algorithm entirely for these. Also add ASCII case-insensitive mode.

### Problem

`src/glob.rs` current flow: every call goes through the full iterative two-pointer
algorithm, even for trivial patterns like `**/*.rs`. ms/edit's `fast_path()` reduces
the common case to a single suffix comparison.

### New `fast_path()` in `src/glob.rs`

Add before `glob_match_inner`, called first by `glob_match`:

```rust
/// Fast path for the two most common glob patterns:
///   `**/*.ext`   — any file with a given extension anywhere in the tree
///   `**/name`    — any file with a specific name anywhere in the tree
///
/// Returns `Some(bool)` if the pattern is handled here, `None` to fall through.
fn fast_path(pat: &[u8], name: &[u8]) -> Option<bool> {
    // Both patterns begin with "**/"
    let suffix = pat.strip_prefix(b"**/")?;
    if suffix.is_empty() || suffix.contains(&b'*') {
        return None; // not a simple suffix pattern
    }

    // `**/*.ext`: suffix starts with `*` → pure extension match
    let (needs_separator, suffix) = if let Some(s) = suffix.strip_prefix(b"*") {
        (false, s)
    } else {
        (true, suffix)
    };

    if suffix.is_empty() || suffix.contains(&b'*') {
        return None;
    }

    // The path must end with `suffix` (case-insensitive, ASCII only)
    if name.len() < suffix.len() {
        return Some(false);
    }
    let name_tail = &name[name.len() - suffix.len()..];
    if !name_tail.eq_ignore_ascii_case(suffix) {
        return Some(false);
    }

    if needs_separator {
        // `**/name`: the character before the suffix must be `/` or it IS the whole name
        let ok = name.len() == suffix.len()
            || name[name.len() - suffix.len() - 1] == b'/';
        Some(ok)
    } else {
        // `**/*.ext`: simple suffix match is sufficient
        Some(true)
    }
}

/// Public entry point — tries fast_path first, falls through to full algorithm.
pub fn glob_match(pattern: &str, path: &str) -> bool {
    let pat = pattern.as_bytes();
    let name = path.as_bytes();
    fast_path(pat, name).unwrap_or_else(|| glob_match_inner(pat, name))
}
```

### Case-Insensitive Variant

Add a second public function for callers that need Windows-compatible matching
(useful when zedit runs on WSL or processes paths from Windows tools):

```rust
/// Like `glob_match` but ASCII case-insensitive.
/// Used for Windows path matching and `.gitignore`-style case folding.
pub fn glob_match_icase(pattern: &str, path: &str) -> bool {
    // Lowercase both sides and delegate. Only allocates on non-ASCII (rare).
    let pat_lower = pattern.to_ascii_lowercase();
    let path_lower = path.to_ascii_lowercase();
    glob_match(&pat_lower, &path_lower)
}
```

| Pattern | Before | After | Speedup |
|---------|--------|-------|---------|

| `a*b*c*d*e*` (complex) | Same | Falls through to full algo | 0× (unchanged) |

## Phase 33: Color Hash Cache in Renderer

**Goal**: Memoize RGB→IndexedColor conversions inside the render hot path using a
256-slot hash table. Prevents repeated OKLab computation for colors that appear
on many cells (which is always the case — a theme typically has < 20 distinct colors).

### Problem

`src/render.rs` current flow (simplified):

```rust
// Called once per changed cell per frame:
fn color_to_ansi16(r: u8, g: u8, b: u8) -> u8 {
    // Calls perceptual_distance() 16 times → 16 × srgb_to_oklab() calls
    // = 16 × ~75 ns = ~1.2 μs per color lookup
    ...
}
```

A 120-column × 40-row terminal = 4 800 cells. On a full redraw with 5 distinct
colors, `color_to_ansi16` is called ~4 800 times but only 5 distinct values
exist. Without caching: 4 800 × 1.2 μs = 5.76 ms just for color mapping.

### New: `ColorCache` in `src/render.rs`

```rust
/// Simple direct-mapped cache: 256 slots, indexed by a fast hash of the RGB value.
/// A slot holds the input key and the computed result; mismatched key = miss.
struct ColorCache {
    keys:   [u32; 256],   // packed RGB: r | (g << 8) | (b << 16)
    values: [u8; 256],    // computed ANSI index (0–15 or 0–255)
    valid:  [bool; 256],  // whether slot has been written
}

impl ColorCache {
    const fn new() -> Self {
        Self { keys: [0; 256], values: [0; 256], valid: [false; 256] }
    }

    #[inline]
    fn slot(r: u8, g: u8, b: u8) -> usize {
        // Knuth multiplicative hash, top 8 bits
        let key = r as u32 | ((g as u32) << 8) | ((b as u32) << 16);
        let h = key.wrapping_mul(2_654_435_761) >> 24;
        h as usize
    }

    #[inline]
    fn get(&self, r: u8, g: u8, b: u8) -> Option<u8> {
        let s = Self::slot(r, g, b);
        let key = r as u32 | ((g as u32) << 8) | ((b as u32) << 16);
        if self.valid[s] && self.keys[s] == key { Some(self.values[s]) } else { None }
    }

    #[inline]
    fn set(&mut self, r: u8, g: u8, b: u8, idx: u8) {
        let s = Self::slot(r, g, b);
        let key = r as u32 | ((g as u32) << 8) | ((b as u32) << 16);
        self.keys[s] = key;
        self.values[s] = idx;
        self.valid[s] = true;
    }
}
```

## Phase 34: SIMD Newline Scan with Runtime Dispatch

**Goal**: Replace the 8-byte SWAR word scan in `scan_newlines()` with vectorized
implementations that process 32–128 bytes per loop iteration, using runtime
CPU feature detection to dispatch to the best available implementation.

### Problem

`src/buffer.rs` current `scan_newlines()`:

```rust
// SWAR: 8 bytes/iteration
while i + 8 <= data.len() {
    let word = u64::from_le_bytes(data[i..i+8].try_into().unwrap());
    let xored = word ^ 0x0A0A_0A0A_0A0A_0A0Au64;
    let has_zero = (xored.wrapping_sub(0x0101_0101_0101_0101u64)) & !xored
                   & 0x8080_8080_8080_8080u64;
    ...
    i += 8;
}
```

ms/edit processes **128 bytes per iteration** on AVX2 (4 × 32 bytes, unrolled)
with `_mm256_cmpeq_epi8` + `_mm256_sad_epu8` for horizontal count. On NEON
(aarch64) it processes 64 bytes per iteration.

### New File: `src/simd.rs`

```rust
//! SIMD-accelerated newline scanning.
//!
//! Runtime dispatch: on first call, detect CPU features and cache a function
//! pointer. All subsequent calls go directly to the best implementation.
//!
//! Throughput:
//!   AVX2  (x86_64):  ~128 bytes / iteration (~32x over byte-by-byte)
//!   NEON  (aarch64): ~64 bytes  / iteration (~16x over byte-by-byte)
//!   SWAR  (fallback): 8 bytes   / iteration  (~4x over byte-by-byte)

/// Count newlines in `data[offset..]` and push each line-start offset
/// (= position after the `\n`) into `out`. `base` is added to every offset.
pub fn scan_newlines(data: &[u8], base: usize, out: &mut Vec<usize>) {
    #[cfg(target_arch = "x86_64")]
    {
        // Runtime dispatch: replace the function pointer on first call.
        use std::sync::atomic::{AtomicPtr, Ordering};
        use std::mem;

        type Fn = fn(&[u8], usize, &mut Vec<usize>);

        static DISPATCH: AtomicPtr<()> = AtomicPtr::new(detect as *mut ());

        fn detect(data: &[u8], base: usize, out: &mut Vec<usize>) {
            let f: Fn = if is_x86_feature_detected!("avx2") {
                scan_avx2
            } else {
                scan_swar
            };
            DISPATCH.store(f as *mut (), Ordering::Relaxed);
            f(data, base, out);
        }

        let ptr = DISPATCH.load(Ordering::Relaxed);
        let f: Fn = unsafe { mem::transmute(ptr) };
        return f(data, base, out);
    }

    #[cfg(target_arch = "aarch64")]
    return scan_neon(data, base, out);

    #[cfg(not(any(target_arch = "x86_64", target_arch = "aarch64")))]
    scan_swar(data, base, out);
}

// ---------------------------------------------------------------------------
// AVX2 implementation (x86_64)
// ---------------------------------------------------------------------------

#[cfg(target_arch = "x86_64")]
#[target_feature(enable = "avx2")]
unsafe fn scan_avx2_inner(data: &[u8], base: usize, out: &mut Vec<usize>) {
    use std::arch::x86_64::*;

    let lf = unsafe { _mm256_set1_epi8(b'\n' as i8) };
    let mut i = 0usize;

    // Align to 32-byte boundary
    let align = data.as_ptr().align_offset(32).min(data.len());
    while i < align {
        if data[i] == b'\n' { out.push(base + i + 1); }
        i += 1;
    }

    // Main loop: 128 bytes per iteration (4 × 32), unrolled for throughput
    while i + 128 <= data.len() {
        let v1 = unsafe { _mm256_loadu_si256(data.as_ptr().add(i)      as *const _) };
        let v2 = unsafe { _mm256_loadu_si256(data.as_ptr().add(i + 32) as *const _) };
        let v3 = unsafe { _mm256_loadu_si256(data.as_ptr().add(i + 64) as *const _) };
        let v4 = unsafe { _mm256_loadu_si256(data.as_ptr().add(i + 96) as *const _) };

        // Each comparison sets matching bytes to 0xFF (== -1 in i8).
        // Subtracting 0xFF from 0 = +1, so we accumulate counts.
        let mut sum = unsafe { _mm256_setzero_si256() };
        sum = unsafe { _mm256_sub_epi8(sum, _mm256_cmpeq_epi8(v1, lf)) };
        sum = unsafe { _mm256_sub_epi8(sum, _mm256_cmpeq_epi8(v2, lf)) };
        sum = unsafe { _mm256_sub_epi8(sum, _mm256_cmpeq_epi8(v3, lf)) };
        sum = unsafe { _mm256_sub_epi8(sum, _mm256_cmpeq_epi8(v4, lf)) };

        // `_mm256_sad_epu8` sums absolute differences against 0 → horizontal sum
        let count_vec = unsafe { _mm256_sad_epu8(sum, _mm256_setzero_si256()) };

        // If any newlines exist in this chunk, scan individually (rare)
        let count = unsafe {
            let hi = _mm256_extracti128_si256::<1>(count_vec);
            let lo = _mm256_castsi256_si128(count_vec);
            let s  = _mm_add_epi64(lo, hi);
            _mm_cvtsi128_si64(s) as u64
        };

        if count > 0 {
            for j in 0..128 {
                if data[i + j] == b'\n' { out.push(base + i + j + 1); }
            }
        }
        i += 128;
    }

    // 32-byte tail
    while i + 32 <= data.len() {
        let v = unsafe { _mm256_loadu_si256(data.as_ptr().add(i) as *const _) };
        let c = unsafe { _mm256_cmpeq_epi8(v, lf) };
        let mask = unsafe { _mm256_movemask_epi8(c) } as u32;
        if mask != 0 {
            let mut m = mask;
            while m != 0 {
                let bit = m.trailing_zeros() as usize;
                out.push(base + i + bit + 1);
                m &= m - 1;
            }
        }
        i += 32;
    }

    // Scalar remainder
    while i < data.len() {
        if data[i] == b'\n' { out.push(base + i + 1); }
        i += 1;
    }
}

#[cfg(target_arch = "x86_64")]
fn scan_avx2(data: &[u8], base: usize, out: &mut Vec<usize>) {
    // SAFETY: only called after `is_x86_feature_detected!("avx2")` is true
    unsafe { scan_avx2_inner(data, base, out) }
}

#[cfg(target_arch = "aarch64")]
fn scan_neon(data: &[u8], base: usize, out: &mut Vec<usize>) {
    use std::arch::aarch64::*;

    let mut i = 0usize;
    let lf = unsafe { vdupq_n_u8(b'\n') };

    // Align to 16 bytes
    let align = data.as_ptr().align_offset(16).min(data.len());
    while i < align {
        if data[i] == b'\n' { out.push(base + i + 1); }
        i += 1;
    }

    // 64 bytes per iteration (4 × 16), unrolled
    while i + 64 <= data.len() {
        let v1 = unsafe { vld1q_u8(data.as_ptr().add(i)) };
        let v2 = unsafe { vld1q_u8(data.as_ptr().add(i + 16)) };
        let v3 = unsafe { vld1q_u8(data.as_ptr().add(i + 32)) };
        let v4 = unsafe { vld1q_u8(data.as_ptr().add(i + 48)) };

        let mut sum = unsafe { vdupq_n_u8(0) };
        sum = unsafe { vsubq_u8(sum, vceqq_u8(v1, lf)) };
        sum = unsafe { vsubq_u8(sum, vceqq_u8(v2, lf)) };
        sum = unsafe { vsubq_u8(sum, vceqq_u8(v3, lf)) };
        sum = unsafe { vsubq_u8(sum, vceqq_u8(v4, lf)) };

        let count = unsafe { vaddvq_u8(sum) };
        if count > 0 {
            for j in 0..64 {
                if data[i + j] == b'\n' { out.push(base + i + j + 1); }
            }
        }
        i += 64;
    }

    // Scalar remainder
    while i < data.len() {
        if data[i] == b'\n' { out.push(base + i + 1); }
        i += 1;
    }
}

// SWAR fallback (any architecture, no intrinsics)
// ---------------------------------------------------------------------------

fn scan_swar(data: &[u8], base: usize, out: &mut Vec<usize>) {
    // Existing implementation from Phase 25 — unchanged.
    const NL_MASK: u64 = 0x0A0A_0A0A_0A0A_0A0A_u64;
    const LO:      u64 = 0x0101_0101_0101_0101_u64;
    const HI:      u64 = 0x8080_8080_8080_8080_u64;

    let mut i = 0;
    while i < data.len() && data.as_ptr().wrapping_add(i).align_offset(8) != 0 {
        if data[i] == b'\n' { out.push(base + i + 1); }
        i += 1;
    }
    while i + 8 <= data.len() {
        let w = u64::from_le_bytes(data[i..i+8].try_into().unwrap());
        let x = w ^ NL_MASK;
        if (x.wrapping_sub(LO) & !x & HI) != 0 {
            for j in 0..8 {
                if data[i+j] == b'\n' { out.push(base + i + j + 1); }
            }
        }
        i += 8;
    }
    while i < data.len() {
        if data[i] == b'\n' { out.push(base + i + 1); }
        i += 1;
    }
}
```

## Phase 35: Sparse Line Cache Checkpoints

**Goal**: Replace the dense `Vec<usize>` line cache (one entry per line) with a
sparse checkpoint array (one entry per 8 192 lines). For lookup, binary-search to
the nearest checkpoint then use the SIMD scan (Phase 34) to reach the target line.

### Problem

`src/buffer.rs` current line cache:

```rust
lines: Vec<usize>,  // byte offset of EVERY line start
```

Memory cost: 8 bytes × number of lines. A 1-million-line file costs 8 MB just for
the cache. For a 10-million-line log file: 80 MB. The cache grows proportionally
with file size, defeating the mmap optimization added in Phase 29.

ms/edit stores one checkpoint per 64 000 lines. zedit's sweet spot is 8 192 (a
power of two that fits in L1 cache for files up to ~65 K lines before needing
heap storage).

### New Data Structure

```rust
/// One cache entry: the byte offset and line index at a checkpoint.
#[derive(Clone, Copy)]
struct LineCheckpoint {
    byte_off: usize,  // byte offset of the start of `line`
    line:     usize,  // line number (0-based)
}

/// Sparse line cache: one checkpoint every CHECKPOINT_STRIDE lines.
/// Between checkpoints, the SIMD scanner finds the exact position.
pub struct LineCache {
    checkpoints: Vec<LineCheckpoint>,
    /// Total number of lines in the buffer (maintained incrementally).
    pub total_lines: usize,
}

const CHECKPOINT_STRIDE: usize = 8_192;
```

Replace the `lines: Vec<usize>` field in `Buffer` with `line_cache: LineCache`.

### New `LineCache` Methods

```rust
impl LineCache {
    pub fn new() -> Self {
        Self {
            checkpoints: vec![LineCheckpoint { byte_off: 0, line: 0 }],
            total_lines: 1,
        }
    }

    /// Full rebuild from a contiguous byte slice.
    /// Called once at file load (mmap or Vec).
    pub fn rebuild(&mut self, data: &[u8]) {
        self.checkpoints.clear();
        self.checkpoints.push(LineCheckpoint { byte_off: 0, line: 0 });
        self.total_lines = 1;

        let mut line = 0usize;
        for (i, &b) in data.iter().enumerate() {
            if b == b'\n' {
                line += 1;
                if line % CHECKPOINT_STRIDE == 0 {
                    self.checkpoints.push(LineCheckpoint { byte_off: i + 1, line });
                }
            }
        }
        self.total_lines = line + 1;
    }

    /// Incremental update after an edit at `edit_byte`.
    /// `delta` is positive for insert, negative for delete.
    /// `newlines_delta` is the net change in newline count.
    pub fn update(&mut self, edit_byte: usize, delta: isize, newlines_delta: isize) {
        // Adjust all checkpoints after the edit point
        let pivot = self.checkpoints.partition_point(|cp| cp.byte_off <= edit_byte);
        for cp in &mut self.checkpoints[pivot..] {
            cp.byte_off = cp.byte_off.wrapping_add_signed(delta);
            cp.line     = cp.line.wrapping_add_signed(newlines_delta);
        }
        self.total_lines = self.total_lines.wrapping_add_signed(newlines_delta);
        // NOTE: Does not re-insert checkpoints for new content — a periodic
        // compaction pass (triggered when checkpoint density degrades too much)
        // handles that. Simple edits do not need perfect checkpoint density.
    }

    /// Find the byte offset of line `target_line`.
    /// Uses binary search on checkpoints, then the SIMD scanner to walk from there.
    pub fn line_to_byte<F>(&self, target_line: usize, scan_fn: F) -> Option<usize>
    where
        F: Fn(usize, usize, usize) -> (usize, usize),
        // scan_fn(from_byte, from_line, to_line) -> (byte_at_to_line, actual_line)
    {
        if target_line >= self.total_lines { return None; }

        // Find the checkpoint just before or at target_line
        let idx = self.checkpoints.partition_point(|cp| cp.line <= target_line);
        let cp  = if idx == 0 { self.checkpoints[0] } else { self.checkpoints[idx - 1] };

        if cp.line == target_line {
            return Some(cp.byte_off);
        }

        // Walk forward from the checkpoint using the SIMD scanner
        let (byte_off, _) = scan_fn(cp.byte_off, cp.line, target_line);
        Some(byte_off)
    }

    /// Convert a byte offset to a line number.
    /// Binary search on checkpoints + forward scalar scan within the stride.
    pub fn byte_to_line<F>(&self, target_byte: usize, count_newlines_fn: F) -> usize
    where
        F: Fn(usize, usize) -> usize,
        // count_newlines_fn(from_byte, to_byte) -> newline count
    {
        let idx = self.checkpoints.partition_point(|cp| cp.byte_off <= target_byte);
        let cp  = if idx == 0 { self.checkpoints[0] } else { self.checkpoints[idx - 1] };
        cp.line + count_newlines_fn(cp.byte_off, target_byte)
    }
}
```

### Integration with `Buffer`

The scanning closures passed to `line_to_byte` and `byte_to_line` delegate to
`crate::simd::scan_newlines` (Phase 34) or the mmap bytes, keeping buffer's
dual-backing-store architecture.

### Memory Comparison

| File size | Lines | Dense cache | Sparse cache (8192 stride) | Saving |
|-----------|-------|-------------|---------------------------|--------|
| 100 KB | ~2 000 | 16 KB | 8 bytes (0 checkpoints) | 100% |
| 10 MB | ~200 000 | 1.6 MB | 3.1 KB (25 checkpoints) | 99.8% |
| 1 GB | ~20 000 000 | 160 MB | 300 KB (2441 checkpoints) | 99.8% |

### Known Issue

Incremental `update()` does not re-insert checkpoints for new content (e.g., pasting
10 000 lines). This degrades lookup performance proportionally to how many lines were
inserted between the last two checkpoints. A compaction pass (rebuild checkpoints in
the affected range) should be triggered when a checkpoint's covered range exceeds
`2 × CHECKPOINT_STRIDE`. This compaction is deferred to the first idle frame.

## Phase 36: Virtual Memory Gap Buffer ✅ DONE

**Goal**: Replace `Vec<u8>` as the gap buffer backing store with a
virtual-memory region: reserve up to 2 GB of address space (using
`mmap(MAP_ANONYMOUS | MAP_PRIVATE, PROT_NONE)`) and commit physical
pages on demand via `mprotect(PROT_READ|WRITE)`. This removes the ~50 MB
practical ceiling on editable file size and eliminates all `Vec::resize()`
reallocations.

### Problem

`src/buffer.rs` current gap buffer:

```rust
data: Vec<u8>
```

When the gap needs to grow, `Vec::resize()` may allocate a new block and
`memcpy` the entire buffer. For a 200 MB file this is a 200 MB copy triggered
mid-keystroke. Additionally, `Vec` commits all physical memory upfront —
opening a 200 MB file for editing allocates 200 MB of RAM immediately.

ms/edit resolves this with its `BackingBuffer::VirtualMemory` path:
- Reserve 4 GB (64-bit) of virtual address space at startup — costs nothing in RAM.
- As the buffer grows, commit 64 KB pages via `mprotect`.
- The buffer never moves in memory — gap moves are `ptr::copy` inside the same
  reserved region.

### New File: `src/vmem.rs`

```rust
//! Virtual memory region: reserve a large address space, commit pages on demand.
//!
//! API mirrors stdext's virtual_reserve/virtual_commit from microsoft/edit.
//!
//! SAFETY: Unix only (Linux, macOS). Uses mmap + mprotect.

use std::io;
use std::ptr::NonNull;

/// Size of each commit chunk. 64 KB matches modern page granularity.
pub const COMMIT_CHUNK: usize = 64 * 1024;

/// Virtual memory region. Memory is reserved but not committed until `grow()`.
pub struct VirtualRegion {
    base:     NonNull<u8>,
    reserved: usize,   // total virtual bytes reserved
    committed: usize,  // total physical bytes committed
}

impl VirtualRegion {
    /// Reserve `size` bytes of virtual address space. No RAM is allocated.
    pub fn reserve(size: usize) -> io::Result<Self> {
        let ptr = unsafe {
            libc::mmap(
                std::ptr::null_mut(),
                size,
                libc::PROT_NONE,            // no access until committed
                libc::MAP_PRIVATE | libc::MAP_ANONYMOUS,
                -1,
                0,
            )
        };
        if ptr == libc::MAP_FAILED {
            return Err(io::Error::last_os_error());
        }
        Ok(Self {
            base: unsafe { NonNull::new_unchecked(ptr as *mut u8) },
            reserved: size,
            committed: 0,
        })
    }

    /// Commit at least `needed` bytes of physical memory from the start.
    /// Pages are committed in `COMMIT_CHUNK` increments to amortize syscalls.
    pub fn grow(&mut self, needed: usize) -> io::Result<()> {
        if needed <= self.committed { return Ok(()); }
        if needed > self.reserved   { return Err(io::Error::from_raw_os_error(libc::ENOMEM)); }

        let new_committed = (needed + COMMIT_CHUNK - 1) & !(COMMIT_CHUNK - 1);
        let new_committed = new_committed.min(self.reserved);

        let ptr = unsafe { self.base.add(self.committed) };
        let len = new_committed - self.committed;
        let rc = unsafe { libc::mprotect(ptr.as_ptr() as *mut libc::c_void, len,
                                         libc::PROT_READ | libc::PROT_WRITE) };
        if rc != 0 { return Err(io::Error::last_os_error()); }

        self.committed = new_committed;
        Ok(())
    }

    /// Raw pointer to the start of the region.
    #[inline]
    pub fn as_ptr(&self) -> *mut u8 { self.base.as_ptr() }

    pub fn committed(&self) -> usize { self.committed }
}

impl Drop for VirtualRegion {
    fn drop(&mut self) {
        unsafe { libc::munmap(self.base.as_ptr() as *mut libc::c_void, self.reserved); }
    }
}

// SAFETY: The region is private (MAP_PRIVATE). We control all access through &mut.
unsafe impl Send for VirtualRegion {}
unsafe impl Sync for VirtualRegion {}

/// Threshold below which we use the heap (`Vec<u8>`) instead of virtual memory.
/// Small buffers (< 128 KB) avoid the mmap/mprotect syscall overhead.
pub const VMEM_THRESHOLD: usize = 128 * 1024;

/// Maximum virtual address space to reserve for large gap buffers.
#[cfg(target_pointer_width = "64")]
pub const VMEM_RESERVE: usize = 2 * 1024 * 1024 * 1024; // 2 GB
#[cfg(target_pointer_width = "32")]
pub const VMEM_RESERVE: usize = 128 * 1024 * 1024;       // 128 MB
```

### Modified `ensure_gap()` / `enlarge_gap()`

The key difference: `VirtualRegion::grow()` replaces `Vec::resize()`. No memcpy of
the entire buffer — only the bytes from `gap_end` to the new end need to slide right:

```rust
fn enlarge_gap(&mut self, needed: usize) {
    let gap_len = self.gap_end - self.gap_start;
    if gap_len >= needed { return; }

    let new_gap_len = (needed + 4096 - 1) & !(4096 - 1); // round up to 4 KB
    let new_total   = self.text_len + new_gap_len;

    match &mut self.storage {
        GapStorage::Heap(v) => {
            // Existing path: shift tail right, resize vec
            let tail_len = self.text_len - self.gap_start;
            v.resize(new_total, 0);
            v.copy_within(self.gap_end..self.gap_end + tail_len,
                          self.gap_start + new_gap_len);
            self.gap_end = self.gap_start + new_gap_len;
        }
        GapStorage::Virtual(region) => {
            // Grow virtual region (commits new pages, buffer does NOT move)
            if region.grow(new_total).is_err() { return; }
            let ptr = region.as_ptr();
            let tail_len = self.text_len - self.gap_start;
            // Slide tail right within the same virtual region
            unsafe {
                std::ptr::copy(
                    ptr.add(self.gap_end),
                    ptr.add(self.gap_start + new_gap_len),
                    tail_len,
                );
            }
            self.gap_end = self.gap_start + new_gap_len;
        }
    }
}
```

### Modified `from_file()`

```rust
pub fn from_file(path: &Path) -> Result<Buffer, String> {
    let file_size = fs::metadata(path).map(|m| m.len() as usize).unwrap_or(0);

    // Large file (> MMAP_THRESHOLD): mmap read-only, line cache from mmap bytes.
    // The gap buffer starts as virtual memory but remains empty until first edit.
    if file_size > MMAP_THRESHOLD {
        if let Ok(map) = crate::mmap::Mmap::open(path) {
            let bytes = map.as_bytes();
            if std::str::from_utf8(bytes).is_err() {
                return Err("File is not valid UTF-8".to_string());
            }
            let mut line_cache = LineCache::new();
            line_cache.rebuild(bytes);

            // Reserve virtual memory for future editing
            let storage = if file_size > crate::vmem::VMEM_THRESHOLD {
                match crate::vmem::VirtualRegion::reserve(crate::vmem::VMEM_RESERVE) {
                    Ok(region) => GapStorage::Virtual(region),
                    Err(_)     => GapStorage::Heap(vec![0u8; INITIAL_GAP]),
                }
            } else {
                GapStorage::Heap(vec![0u8; INITIAL_GAP])
            };

            return Ok(Buffer {
                storage,
                gap_start: 0,
                gap_end:   INITIAL_GAP,
                text_len:  0,       // still backed by mmap, gap buffer empty
                line_cache,
                modified:  false,
                file_path: Some(normalize_to_relative(path)),
                mmap:      Some(map),
            });
        }
    }

    // Small file: read into heap Vec (< VMEM_THRESHOLD) or virtual (>= VMEM_THRESHOLD)
    let content = fs::read(path).map_err(|e| format!("Failed to read: {e}"))?;
    let content_len = content.len();

    let mut storage = if content_len >= crate::vmem::VMEM_THRESHOLD {
        match crate::vmem::VirtualRegion::reserve(crate::vmem::VMEM_RESERVE) {
            Ok(mut region) => {
                let _ = region.grow(content_len + INITIAL_GAP);
                GapStorage::Virtual(region)
            }
            Err(_) => GapStorage::Heap(Vec::with_capacity(content_len + INITIAL_GAP)),
        }
    } else {
        GapStorage::Heap(Vec::with_capacity(content_len + INITIAL_GAP))
    };

    // Copy content into storage
    match &mut storage {
        GapStorage::Heap(v) => {
            v.extend_from_slice(&content);
            v.resize(content_len + INITIAL_GAP, 0);
        }
        GapStorage::Virtual(region) => {
            unsafe {
                std::ptr::copy_nonoverlapping(content.as_ptr(), region.as_ptr(), content_len);
            }
        }
    }

    let mut line_cache = LineCache::new();
    line_cache.rebuild(&content);

    Ok(Buffer {
        storage,
        gap_start:  content_len,
        gap_end:    content_len + INITIAL_GAP,
        text_len:   content_len,
        line_cache,
        modified:   false,
        file_path:  Some(normalize_to_relative(path)),
        mmap:       None,
    })
}
```

### Known Issues to Resolve Before Implementation

| Issue | Resolution |
|-------|-----------|
| `materialize_mmap()` must grow the virtual region before copying | Call `region.grow(mmap_len + INITIAL_GAP)` before `ptr::copy_nonoverlapping` |
| `byte_at()` must handle `GapStorage::Virtual` raw pointer | Use `unsafe { *ptr.add(physical_pos) }` |
| `text_bytes()` (for save) must still produce a contiguous slice | `extract_raw()` pattern: two slices before and after gap, collected into `Vec<u8>` |
| mprotect is slow on macOS for small regions | Use `COMMIT_CHUNK = 2 MB` on macOS; 64 KB on Linux |
| `virtual_reserve(2GB)` fails in 32-bit containers | Fall back to `GapStorage::Heap` if reserve fails |

### Expected Benefit

| Scenario | Before | After |
|----------|--------|-------|
| Open 500 MB file for editing | mmap only, blocks on first edit (500 MB copy) | mmap + virtual region; edits commit 64 KB at a time |
| Gap grow during large paste | `Vec::resize()` → full buffer memcpy | `mprotect` on new pages only + in-place slide |
| RAM usage browsing 1 GB log | ~1 GB (mmap forces eventual full RAM) | OS-paged: only accessed pages in RAM |
| Practical editable size limit | ~50 MB | ~2 GB (virtual reserve) |

## Design Constraints (Inherited)

- Zero external Rust crate dependencies — all new code uses `std` + libc FFI only.
- `src/vmem.rs` uses `libc::mmap` + `libc::mprotect` — same pattern as `terminal.rs`.
- `src/simd.rs` uses `std::arch` intrinsics — part of the Rust standard library.
- Startup time budget: < 10 ms (virtual reserve is instant; no physical pages committed).
- Binary size budget: < 1 MB stripped (LUT adds 1 KB; SIMD code adds ~2 KB).
- All user-facing strings remain in English.
